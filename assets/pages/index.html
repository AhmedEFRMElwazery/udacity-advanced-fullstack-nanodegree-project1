Welcome to the Image Processing API

This webpage will serve as a guideline on how to use the Image Processing API

How to use:

You can use the Image Processing API by following these steps:

  1. Add the photo(s) that you want to resize in the assets/images/fullfolder
  2. Visit the following link: http//localhost:1987/api/v1/processimage
  3. Add the following parameters after the URL:
    • ?imageName=[filename] -> replace "filename" with the name of the original image you added to the "full" folder, or you can write "all" to process all the original images in the "full" folder.
    • &desiredWidth=[number] -> replace "number" with the number of pixels you wish to have as your width.
    • &desiredHeight=[number] -> replace "number" with the number of pixels you wish to have as your height.
  4. As an example, the final url should look like this: http//localhost:1987/api/v1/processimage?imageName=santamonica&desiredWidth=900&desiredHeight=700 in the address bar in your browser.
  5. Press "Enter".

What you will get:

  • Processed image(s) with the desired width and height will be stored in assets/images/thumb folder.

  • For ease of reach and navigation, the processed image will be saved in the format of imageName_width_[pixels]_height_[pixels].jpeg.

  • ONLY new processed image(s) with new dimensions (width and height) are the ones that will be created.

  • If you enter resizing parameters that were previously used in an ealier processing, and the processed image is still available in the thumb folder, it will NOT be reprocessed, but rather load the already processed image.

  • If you enter "all" for the "imageName" parameter, all the photos in the "full" folder will be processed to match the values entered for the "desiredWidth" and "desiredHeight" parameters. The last image that is sorted alpha-numerically in the folder will be displayed.

  • A logger file named "loggerFile.txt" is created and stored in the "assets" folder. It saves all the API calls over its lifetime. Details such as date and time of the API call, parameters values entered by the user, remote ip address of the user, URLs, etc., is being stored. If by any chance it gets deleted, a new logger file (i.e. loggerFile.txt) is created automatically once the server is up and running and a call has been made to the API.
